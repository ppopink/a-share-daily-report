# K线选股模型 — 操作文档

## 一、模型概述

从A股沪深主板中，结合**技术面**、**资金流向**、**板块/相对强度**三个维度，每日精选10只最符合短线交易条件的股票。项目路线是：先做规则选股，每天收盘后输出候选；再用严格回测验证；如果规则稳定，再把规则特征喂给模型做预测选股。

### 选股范围

- **沪深主板**：600/601/603/605（上海）、000/001（深圳）
- **排除**：科创板(688)、创业板(300/301)、中小板(002/003)、北交所(4/8/9)、ST股票
- **总量**：约2122只

## 二、技术指标条件

### 必选条件（全部满足才入选）

| 条件 | 参数 | 说明 |
|------|------|------|
| 均线多头 | close > MA3 > MA5 > MA21 | 价格站上短中期均线 |
| ADX趋势 | DMI(7,6)，+DI > -DI 且 ADX > 25 | 趋势向上且强度足够 |
| EXPMA金叉 | EXPMA(7,21)，近3日上穿21日线 | 短期走强 |
| MACD二次金叉 | MACD(6,13,5)，进入0轴上方区域后的第二次金叉，近3日触发 | 经典看涨信号 |
| 放量 | 当日成交量 > 前5日均量 × 1.3 | 有资金进场，分母不含当日量 |
| 成交额 | 当日成交额 > 1亿元 | 过滤流动性不足标的 |
| 资金流 | 主力净流入 > 0 | API失败时用量价估算 |
| 板块 | 板块前20%，个股在板块内前20% | 只选热点中的领涨股 |

### 加分项

| 项目 | 满分 | 说明 |
|------|------|------|
| ADX强度 | 15 | ADX越高趋势越强 |
| MACD质量 | 15 | 金叉角度、DIFF/DEA距离 |
| EXPMA乖离 | 10 | 金叉后乖离率合理 |
| 均线多头 | 10 | MA3 > MA5 > MA21 |
| 量价配合 | 5 | 放量上涨优于放量下跌 |

## 六、回测验证

### 运行回测

```bash
# 快速回测（500只，适合先验证）
python main.py --backtest --test 500

# 完整回测（全量主板，耗时取决于网络和缓存）
python main.py --backtest

# 自定义回测
python main.py --backtest --start 2025-06-01 --end 2026-06-01 --hold 5,10
```

### 回测口径

- 信号日：使用截至当日收盘的K线计算选股信号。
- 买入价：默认下一交易日开盘价买入（`BACKTEST_BUY_MODE = "next_open"`）。
- 卖出价：持有N个交易日后按收盘价卖出。
- 成本：默认买卖合计扣除 `BACKTEST_TRADE_COST_PCT = 0.15%`。
- 排序：完整回测复用实盘选股的综合评分，包括技术、估算资金流、板块/相对强度和入场时机加减分。

### 回测结论

- 优先看平均收益、中位收益、最大亏损和胜率，不要只看累计收益。
- 若胜率不高但平均收益为正，说明策略依赖少数趋势大行情，需要配合止损和仓位控制。
- 调参后应重新跑同一时间段回测，比较不同参数组合，而不是只看单次结果。

## 七、参数调优

所有参数在 `config.py` 中，按需修改：

```python
# 技术指标
ADX_THRESHOLD = 25    # 调高 → 信号减少但更可靠（建议30-35）
VOL_RATIO_MIN = 1.3   # 调高 → 只要真正放量的
EXPMA_CROSS_DAYS = 3  # 只选最新鲜的EXPMA金叉
MACD_CROSS_DAYS = 3   # 只选最新鲜的MACD二次金叉
MIN_AMOUNT = 100_000_000  # 成交额至少1亿元

# 硬过滤开关
REQUIRE_POSITIVE_MONEY_FLOW = True
REQUIRE_SECTOR_HOT = True
REQUIRE_LEADING_IN_SECTOR = True

# 评分权重
WEIGHT_TECHNICAL = 55  # 技术面权重
WEIGHT_CAPITAL_FLOW = 25  # 资金面权重
WEIGHT_SECTOR_LEAD = 20   # 板块权重

# 回测交易口径
BACKTEST_BUY_MODE = "next_open"
BACKTEST_TRADE_COST_PCT = 0.15
```

调参后重新回测对比效果即可找到最优参数。

## 九、常见问题

**Q: 为什么选出的股票这么少？**
A: 四个必选条件同时满足的股票很少（约0.6-1%），这是模型对信号质量的要求。

**Q: 能不能放宽条件？**
A: 修改 `config.py` 中的阈值参数，然后 `python main.py --backtest --test 500` 对比回测结果。

**Q: 周末运行会怎样？**
A: 自动检测非交易日，推送最近一个周五的推荐。

**Q: 资金流向数据总是API失败？**
A: 东方财富API有频率限制，模型会自动用K线量价关系估算资金方向作为备选。
